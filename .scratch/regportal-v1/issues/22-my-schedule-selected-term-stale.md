## Bug

`selectedTermId` in `ScheduleView` is never reset when the term it points to is removed from `registrationTerms`. If a student manually selects a term from the selector and then drops all courses in that term, `registrationTerms` recomputes to exclude the now-empty term, hiding the selector. But `selectedTermId` remains set, so `resolvedTermId` keeps returning the stale term instead of falling back to `defaultTermId`. The grid and summary list show empty forever, with no UI control to switch terms.

## Agent Brief

**Category:** bug
**Summary:** selectedTermId goes stale after dropping the last course in a manually-selected term — student locked on empty schedule view

**Current behavior:**
`resolvedTermId = computed(() => selectedTermId.value ?? defaultTermId.value)`. When the user picks a term via the selector, `selectedTermId` is set to a non-null value. If they then drop all courses in that term, `registrationTerms` shrinks and the term selector is hidden (only shown when `registrationTerms.length > 1`). `selectedTermId` is never cleared, so `resolvedTermId` still returns the empty term. The student sees "No registrations" with no way to navigate to their remaining courses except a full page reload.

**Desired behavior:**
When `registrationTerms` changes such that `selectedTermId` no longer refers to a term in that list, `selectedTermId` should be reset to `null` so that `resolvedTermId` falls back to `defaultTermId` (the D-status term with remaining courses).

**Key interfaces:**
- `ScheduleView` setup — add a `watch(registrationTerms, ...)` that sets `selectedTermId.value = null` when the currently selected term is no longer present in the list
- `registrationTerms` computed and `selectedTermId` ref — no signature changes needed
- `resolvedTermId` computed — no changes needed; the `?? defaultTermId.value` fallback already handles null correctly

**Acceptance criteria:**
- [ ] After dropping all courses in a manually-selected term, `selectedTermId` resets to null
- [ ] After the reset, `resolvedTermId` returns the D-status term (or the first remaining eligible term)
- [ ] The grid and summary list show the student's remaining courses for the fallback term
- [ ] If the student has only one term (no manual selection was made), behavior is unchanged
- [ ] A unit test covers: two terms → user selects term B → drops all B courses → view shows term A's courses

**Out of scope:**
- Resetting `selectedTermId` for reasons other than the term becoming empty
- Animations or transition effects for the term switch

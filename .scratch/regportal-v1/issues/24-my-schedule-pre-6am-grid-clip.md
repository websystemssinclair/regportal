## Bug

`toGridPercent` in `ScheduleView` returns a negative value for any section that starts before 6 AM (`GRID_START_MINUTES = 360`). A section at 5:30 AM produces `top: -2.78%`, which positions the block above the grid container. The container clips it (`overflow-hidden`), making the section invisible with no indication it exists.

## Agent Brief

**Category:** bug
**Summary:** Sections starting before 6 AM receive a negative top% and are rendered above the visible grid area

**Current behavior:**
`toGridPercent(minutes)` returns `((minutes - 360) / 1080) * 100`. For any `minutes < 360` (before 6 AM), the result is negative. Blocks with a negative `top` value are positioned above the grid container, which has `overflow-hidden`, so they are invisible. No fallback, clamping, or "out-of-range" indicator exists.

**Desired behavior:**
Two acceptable fixes — pick whichever fits best after reading the current implementation:
1. **Clamp**: Apply `Math.max(0, toGridPercent(startMin))` so out-of-range blocks appear at the top of the grid rather than disappearing. Similarly clamp the bottom via height.
2. **Extend the range**: Lower `GRID_START_MINUTES` to 5 AM (300) or an earlier bound so early-morning sections are visible. Update any time-axis labels if they exist.

The key invariant: a section with a valid `StartTime` must be visible somewhere in the grid, not silently clipped.
